# 收入报警功能说明

## 当前状态
✅ 收入报警功能已移除

## 移除原因
计划创建专门的报警机器人，因此从日报生成脚本中移除了临时报警功能。

## 之前的功能（已移除）
- 自动检测收入下降超过30%
- 在日报中添加醒目的报警提示
- 显示前日、昨日收入和下降金额

## 未来计划
创建专门的报警机器人，可以：
- 支持@所有人或@特定成员
- 触发推送通知
- 独立于日报运行
- 可以设置多个报警阈值

## 当前日报格式
```
**昨日（2026-02-05）总览数据**
- DAU：36,010
- 新增用户：1,332
- 总收入：$23,689.53
- 付费用户数：792
- 付费率：2.20%
- ARPU：$0.66
- ARPPU：$29.91

对照前日（2026-02-04）变化：
- DAU：37,245 → 36,010 (-1,235, -3.32%)
- 新增用户：1,336 → 1,332 (-4, -0.3%)
- 总收入：$17,432.09 → $23,689.53 (+6,257.44, +35.9%)
- 付费用户数：752 → 792 (+40, +5.32%)
- 付费率：2.02% → 2.20% (+0.18, +8.91%)
- ARPU：$0.47 → $0.66 (+0.19, +40.43%)
- ARPPU：$23.18 → $29.91 (+6.73, +29.03%)

**变化原因细拆：**
- 收入增长29%来自Steam：该渠道收入增长$1,821.16，占总收入增长的29%
- DAU下降80%来自PC官包：该渠道DAU减少921，占总DAU下降的80%
- 收入增长49%来自美国：该国家收入增长$1,813.84，占总收入增长的49%
- DAU下降45%来自日本：该国家DAU减少213，占总DAU下降的45%
```
